# ZW-finShop. Заявка на кредит Онлайн  
## Проектная работа

### I.  Модуль "Банк-клиент"

1. Клиент авторизуется на сайте (логин, пароль)
2. Клиент вводит заполняет свой профиль: 
    * Имя 
    * Паспорт
    * Доход
3. Клиент вводит запрашиваемую сумму, срок кредита и отправляет заявку
4. Заявка создается в статусе "Ожидает решения"
5. Через некоторое время статус заявки реактивно изменяется (Webflux) в зависимости от принятого Банком решения

Сайт взаимодействует со свойим Сервером (контроллером) по http+ws.
Сервер организует клиентское взаимодействие с Дата-центром по GRPC.  

### II. Модуль "Дата-центр"

1. БД Postgres
2. Шина Kafka (для интеграции с Кредитным Менеджером)
3. Веб Интерфейс GRPC (для интеграции с Банк-Клиентом)


### III. Модуль "Кредитный менеджер"

1. Рабочеее место - сайт. Страница находится в режиме "Ожидание Заявки"
2. При поступлении заявки, реактивно выполняется переход в карточку Заявки     
3. Менеджер одобряет, или отказывает заявку. В случае одобрения, можно указать сумму
4. Сообщение передается клиенту через Дата-центр 

Сайт взаимодействует со свойим Сервером (контроллером) по http+ws.
Сервер организует клиентское взаимодействие с Дата-центром по шине Kafka.  

Все сервисы исполняются в docker-контейнерах.
